## 🗑️ The Architecture Rant (Or: Why Nested Directories Are a Joke)

Before we get into the setup, let's address the single-file structure. Most people—specifically those addicted to standard tutorial structures—think that having multiple nested directories (directories inside directories inside directories) is the "right" way to code. It's not. 

You spread your logic across fifty files, and what happens? You enter relative-import configuration hell. Your code crashes, you're left staring at a screen wondering which of the hundred imports broke, and debugging becomes an archaeological dig. I've built those complex structures before. They work, sure, but they are an absolute waste of time. 

If you actually believe in clean architecture, you want to know exactly where a bug lives. One unified file? I know exactly where the error is. I edit a single line, push the changes, and it's fixed in 60 seconds flat. If you use nested directories, one small change means refactoring imports across your entire repo. 

Anyway, let's get back to the benchmark.

## 💎 The Zero-Risk Architecture & Deployment

I originally planned to host this on my own servers. You know the drill: build a standard application, force you through a tedious sign-up process, collect your user data, and lock the code away behind an API like everyone else does. 

But I realized every developer actually needs this. Gatekeeping it behind a SaaS login is pointless. So, I open-sourced it. I threw the entire architecture onto free cloud providers like Render so anyone can use it without signing up, without logging in, and without giving up a single drop of personal data. 

The entire engine is condensed into exactly **two files**:

1.  **`index.py`**: A unified, self-contained logical matrix. It houses the entire FastAPI backend, the automated semantic evaluation judge system, model routing protocols, and the complete React/Tailwind web user interface embedded inside a single script.
2.  **`deps.txt`**: The dedicated environmental blueprint containing every precise package version needed to spin up the gauntlet.

*(Note: If you notice a `vercel.json` file lying around in the repo, ignore it. It was intended for a previous Vercel deployment, but the architecture is fully running on Render now. You can deploy this anywhere you want.)*

You don't need to look at the source code. You don't need to install any dependencies. You don't even need to clone the repository. ### 💻 Local Setup
1. Clone the repository.
2. Install the verified dependencies:
   ```bash
   pip install -r deps.txt
   ```
3. Set your environment variables:
   ```bash
   export GOOGLE_API_KEY="your-key-here"
   export HF_TOKEN="your-token-here"
   ```
4. Spin up the local FastAPI development server:
   ```bash
   uvicorn index:app --reload
   ```

### UI Navigation
Once running, navigate to `http://localhost:8000` to access the **AGI Systems Directorate Terminal**. Select your target inference model (e.g., `aeterna-vox`), assign a secondary model as the Semantic Judge (e.g., `gemini`), and execute the full gauntlet to view live-streamed stability metrics and evaluation logs.
